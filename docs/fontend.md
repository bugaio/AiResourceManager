# AiResourceManager 前端页面设计文档

## 1. 技术栈

| 项目 | 选型 |
|------|------|
| 框架 | Vue 3 (Composition API + `<script setup>`) |
| 构建 | Vite |
| 语言 | TypeScript |
| 组件库 | Element Plus |
| CSS框架 | Tailwind CSS 3.x |
| 状态管理 | Pinia |
| 编辑器 | Monaco Editor |
| 路由 | Vue Router |
| 主题 | 亮色/暗色切换 |

---

## 2. 整体布局

```
┌─────────────────────────────────────────────────────────┐
│  TopBar (56px)                                          │
│  [Logo/名称]     [  skill | mcp | subAgent  ]   [⚙设置]│
├────────────────┬────────────────────────────────────────┤
│  Sidebar       │  Content Area                          │
│  (可拖拽宽度   │  (自适应剩余空间)                       │
│   默认240px)   │                                        │
│                │                                        │
│  ┌──────────┐  │                                        │
│  │ 分组列表  │  │                                        │
│  │ (自适应)  │  │                                        │
│  ├──────────┤  │                                        │
│  │ 目标路径  │  │                                        │
│  │ (自适应)  │  │                                        │
│  └──────────┘  │                                        │
└────────────────┴────────────────────────────────────────┘
```

- 侧栏宽度：默认240px，可拖拽分割线调整
- 侧栏上下分区：根据内容自动调整高度（非固定比例）
- 整体高度：100vh，内部滚动

---

## 3. 路由结构

```
/                         → 重定向到 /resources
/resources                → 主视图（三栏布局）
/aliases                  → 路径别名管理页
/data                     → 数据导入导出页
```

三栏布局在 `/resources` 路由下常驻，`/aliases` 和 `/data` 为独立全屏路由页面。

## 5. 左侧栏 (Sidebar)

### 5.1 上部分 — 分组列表

#### 顶部操作区

- 标题："分组"
- 右侧齿轮按钮 ⚙，点击弹出 Popover/Dropdown：
  - 新增分组（输入名称）
  - 配置是否默认展开全部分组

#### 分组列表项

```
┌──────────────────────────┐
│ 📁 全部                   │  ← 固定项，始终存在
├──────────────────────────┤
│ 📁 常用工具          •••  │  ← 自定义分组
│ 📁 项目专用          •••  │
│ 📁 实验性            •••  │
└──────────────────────────┘
```

- "全部"：展示当前类型所有资源，固定在顶部
- 自定义分组：右侧 `•••` 按钮，点击弹出菜单：
  - 重命名
  - 部署到目录（选择路径别名或手动输入路径）
  - 删除分组
- 点击分组 → 右侧内容区筛选显示该分组的资源
- 当前选中分组高亮

#### 新增分组弹窗

- 输入框：分组名称（必填）
- 确认/取消按钮

### 5.2 下部分 — 目标路径列表

#### 标题区

- 标题："已部署目标"
- 右侧"检查"按钮：手动触发健康检查（POST /api/v1/deployments/check）

#### 列表内容

根据已注册的路径别名 + 操作历史中的路径显示：

```
┌──────────────────────────────────┐
│ 📂 claude全局 (~/.claude)         │
│   ├ [组🔄] 常用工具          ✕   │  ← 整组引用+track，只能整组移除
│   ├ my-skill                 ✕   │  ← 单个引用，可单个移除
│   ├ ⚠️ broken-skill    [修复|清理] │  ← 链接已断开
│   └ context7-mcp             ✕   │
├──────────────────────────────────┤
│ 📂 /projects/app/.claude          │
│   └ [组] 项目专用            ✕   │
└──────────────────────────────────┘
```

- 整组引用：显示 `[组] 分组名`，有 track 则显示 🔄，只能整组移除
- 单个引用：显示资源名称，可单个移除（✕ 按钮）
- **Broken 状态**：⚠️ 图标 + 橙色文字，操作按钮变为：
  - "修复" → 调 POST /api/v1/deployments/:id/repair 重建链接
  - "清理" → 调 DELETE /api/v1/deployments/:id/items/:item_id 删除记录
- 移除操作需二次确认
- 路径为空（无部署）时不显示该路径

#### 健康检查触发时机

- 每次查看侧栏目标路径列表时（GET /api/v1/targets 返回中含 status 字段）
- 用户点击"检查"按钮时（全量检查）

## 6. 右侧内容区 (Content Area)

### 6.1 顶部操作栏

```
┌─────────────────────────────────────────────────┐
│ 🔍 搜索资源名称...    [+ 新建]  [▦ 网格|☰ 列表] │
├─────────────────────────────────────────────────┤
│ □ 全选  │ 已选3项: [部署到...] [移动到分组] [删除]│  ← 批量操作栏(选中时出现)
└─────────────────────────────────────────────────┘
```

- 搜索框：按名称实时筛选
- 新建按钮：打开新建资源表单
- 视图切换：网格 / 列表 两种模式
- 批量操作栏：当有卡片被选中时，搜索栏下方出现操作栏
  - 部署到指定路径
  - 移动到分组
  - 批量删除（调 DELETE /api/v1/resources/batch，展示关联信息后确认）

左侧栏分组旁也有 `+` 按钮，点击后直接新建资源并归入该分组。

### 6.2 网格视图 — 资源卡片

```
┌────────────────────────────┐
│ □                     •••  │  ← 左上角多选框 + 右上角常驻操作按钮
│                            │
│  skill-name                │  ← 资源名称（加粗）
│  这是一个用于xxx的skill      │  ← 描述（最多2行截断）
│                            │
│  [常用工具] [项目专用]       │  ← 分组标签 (chip)
│                            │
│  📍 claude全局, /proj/app  │  ← 部署状态：已部署路径
│  🕐 2026-06-10 14:30       │  ← 最后修改时间
└────────────────────────────┘
```

- 响应式列数：根据内容区宽度自动调整（3-4列）
- 多选框：左上角，点击选中进入批量模式
- `•••` 操作按钮：常驻右上角，hover时更明显，点击弹出菜单：
  - 编辑内容（打开Monaco编辑器抽屉）
  - 编辑信息（名称、描述）
  - 移动/添加到分组
  - 部署到目录
  - 删除
- hover时卡片有轻微阴影提升效果

### 6.3 列表视图

```
┌───┬──────────┬──────────┬───────────────┬────────────┬─────┐
│ □ │ 名称     │ 描述     │ 分组          │ 部署状态    │ ••• │
├───┼──────────┼──────────┼───────────────┼────────────┼─────┤
│ □ │ my-skill │ 用于xxx  │ 常用工具      │ claude全局  │ ••• │

```

- 表格形式，信息密度高
- 支持列排序（名称、修改时间）
- 同样支持多选和批量操作

## 7. Monaco 编辑器抽屉

### 7.1 触发方式

- 卡片操作菜单 → "编辑内容"
- 双击卡片

### 7.2 抽屉规格

- 方向：从右侧滑出
- 宽度：默认占屏幕70%，可拖拽左边缘调整
- 遮罩：半透明遮罩覆盖剩余区域

### 7.3 抽屉内容

```
┌──────────────────────────────────────┐
│  编辑: my-skill.md          [✕ 关闭] │
├──────────────────────────────────────┤
│                                      │
│  ┌──────────────────────────────┐    │
│  │                              │    │
│  │   Monaco Editor              │    │
│  │   (语法高亮 + 自动补全)       │    │
│  │                              │    │
│  │                              │    │
│  └──────────────────────────────┘    │
│                                      │
├──────────────────────────────────────┤
│              [取消]    [保存]         │
└──────────────────────────────────────┘
```

- 根据资源类型自动设置语言模式：
  - skill / subAgent → Markdown
  - MCP → JSON (支持注释高亮)
- 底部操作栏：取消（丢弃更改）、保存
- 未保存时关闭需确认提示
- 支持 Ctrl+S / Cmd+S 快捷键保存

## 11. 视觉风格与设计规范

### 11.1 整体风格

参考风格：干净、低饱和、卡片式布局。特征：
- 大量留白，内容不拥挤
- 白底卡片 + 浅灰背景形成层次
- 侧栏深色（深灰绿/深灰蓝），内容区浅色
- 圆角柔和，阴影轻微
- 图标简洁（线条风格），文字层级分明

## 14. 分页

### 14.1 分页组件

- 使用 Element Plus 的 `el-pagination` 组件
- 位置：内容区底部
- 显示内容：总条数 + 每页条数选择(10/20/50) + 页码切换

### 14.2 分页参数

列表请求统一携带 `page` 和 `page_size` 参数。响应中包含 `total` 用于计算总页数。

### 14.3 分页状态

存储在各 store 中：
```ts
interface PaginationState {
  page: number       // 当前页码，默认 1
  pageSize: number   // 每页条数，默认 20
  total: number      // 总条数（接口返回）
}
```

切换 Tab、切换分组、搜索时自动重置 page=1。

## 15. WebSocket

### 15.1 连接管理

`src/api/ws.ts` 封装：
- 连接地址：`ws://localhost:3678/api/v1/ws`
- 自动重连：断开后指数退避（1s → 2s → 4s → 8s → 最大30s）
- 心跳保活：每30s发送 ping
- 连接状态暴露给 UI（可选显示连接状态指示器）

### 15.2 事件处理

收到服务端推送后，根据 event 类型执行：

| 事件 | 前端行为 |
|------|---------|
| `resource:updated` | 刷新资源列表（若当前页包含该资源） |
| `resource:deleted` | 从列表中移除该资源，toast提示 |
| `deploy:synced` | 刷新部署列表 + 侧栏目标路径列表，toast提示同步完成 |

### 15.3 与 Store 集成

在 App.vue 的 onMounted 中建立 WebSocket 连接，事件回调中直接调用 store 的 action 刷新数据。
